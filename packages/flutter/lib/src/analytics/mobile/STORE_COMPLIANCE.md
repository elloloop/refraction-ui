# Mobile analytics — App Store / Play Store compliance

Release-blocking deliverable for the `refraction_ui` analytics **mobile
(iOS/Android)** layer (epic #230 §3, issue #232). Zero store-rejection risk is
acceptance criteria. This documents **exactly what is and is not collected**,
the iOS Privacy Manifest contribution, the Google Play Data Safety mapping,
and the ATT/IDFA + consent sequencing.

`refraction_ui` is a **dependency-free Dart UI library**. It ships **no**
native code and bundles **no** Firebase/PostHog/secure-storage SDK. The
network egress, the vendor SDKs, and the durable secure store are all things
the **consuming app** wires in (via the injectable ports in this layer).
Analytics go to the **consumer's** endpoint or the **consumer's** configured
Firebase/PostHog project — **never to refraction-ui's servers** (the only
"report back to us" path is the separate, opt-in error-feedback epic #247).

---

## 1. What is collected (and what is NOT)

### Collected (only after consent — see §4)

| Data | Where it comes from | Identifier linkage | Purpose |
|---|---|---|---|
| **Product interaction**: screen/page views, custom `track` events + their non-PII properties | The app's own `analytics.track/screen/page` calls | The resettable, non-PII **`anonymousId`** (a random UUIDv4) and/or the opaque app-supplied `userId` | Product analytics |
| **Coarse app/session context**: `app`, `env`, library name/version, analytics `sessionId`, `timestamp` | Generated by the library | `anonymousId` | Analytics, sessionisation |
| **Opaque `userId`** (only if the app calls `identify`) | App-supplied; the app owns this record | Itself | Analytics |

PII keys in event properties/traits are **redacted** before any sink sees them
by the existing core `Redactor` (deny-list + caller `redactKeys`).

### NOT collected by this library

- **No IDFA / advertising identifier.** The library never reads
  `ASIdentifierManager`/IDFA. The `tracking` consent category exists only to
  *gate* sinks behind iOS ATT — the identifier itself is never accessed or
  stored here.
- **No device fingerprint, no precise location, no contacts, no
  Android `AAID`** are read by this layer.
- **No data to refraction-ui.** No refraction-ui-owned endpoint.
- **No identifiers before consent** — see §4. Nothing is minted or written to
  disk until the relevant consent category is granted.

> The consuming app may, via the vendor SDKs *it* configures, enable
> additional Firebase/PostHog auto-collection (e.g. Firebase auto screen
> tracking, device model). That collection belongs to the consumer's
> Firebase/PostHog configuration and **must be declared by the consuming app**
> per the `firebase_analytics`/`posthog_flutter` plugins' own manifests and
> their docs. This library only forwards the canonical envelope.

## 2. iOS — Privacy Manifest + required-reason APIs

See [`PrivacyInfo.xcprivacy`](./PrivacyInfo.xcprivacy) (a **fragment** to merge
into the consuming app's `ios/Runner/PrivacyInfo.xcprivacy`).

- `NSPrivacyTracking = false` for this library (no IDFA, no cross-app
  tracking by `refraction_ui` itself).
- **Required-reason API** — offline persistence of the `anonymousId` +
  session via the consumer-injected secure store is *User Defaults*-class:
  declare `NSPrivacyAccessedAPICategoryUserDefaults` with reason **`CA92.1`**
  ("access info from same app, per documentation") — the only data persisted
  is this app's own resettable non-PII id, never shared with another app.
- `NSPrivacyCollectedDataTypeProductInteraction`, **not linked to identity,
  not used for tracking**, purpose = Analytics.
- The consuming app must **also** carry the manifests shipped by the
  `firebase_analytics`, `posthog_flutter`, and `flutter_secure_storage` /
  `shared_preferences` plugins it adds (each plugin ships its own).

## 3. Android — Google Play Data Safety mapping

Declare in the Play Console **Data safety** form (for the data this library's
flow produces; add the consumer's own additions):

| Play Data Safety field | Value |
|---|---|
| Data type | **App activity → App interactions / Other user-generated content** (screen/track events); **App info & performance → Other (session/diagnostic context)** |
| Collected? | Yes (only after in-app consent) |
| Shared? | Shared **only** with the data processor the *consumer* configures (their backend / their Firebase / their PostHog). Not sold. Not shared with refraction-ui. |
| Ephemeral? | No (the resettable `anonymousId` is persisted to enable sessionisation) |
| Required or optional? | **Optional** — gated by the app's consent UI; collection does not start until granted |
| Encrypted in transit? | Yes (consumer endpoint must be HTTPS; Firebase/PostHog SDKs use TLS) |
| User can request deletion? | Yes — `analytics.reset()` mints a fresh `anonymousId`; `MobileConsentController.revokeAll()` purges the persisted identity and stops collection |
| Purpose | Analytics |
| Linked to identity? | Only to the opaque app `userId` if the app calls `identify`; otherwise pseudonymous |
| Advertising ID (AAID) | **Not collected by this library** |

The persistent store uses Android Keystore-backed
`EncryptedSharedPreferences` / `flutter_secure_storage` **as supplied by the
consumer** (injected via `SecureKeyValueStore`); nothing is bundled.

## 4. ATT / IDFA + consent sequencing (enforced in code)

Implemented by `MobileConsentController` + `ConsentGatedSecureStorage`
(`consent_gate.dart`, `secure_storage.dart`). Per-sink consent categories are
enforced by the **existing** core router gate (`Consent.allows`) — no API/
contract change; the sinks just declare `consentCategories`.

Mandatory ordered flow:

1. **Boot — nothing granted.** No consent categories are granted; the gated
   secure store returns empty and drops writes ⇒ **no `anonymousId`/session is
   minted or written to disk before consent**. ATT is **not** auto-prompted.
2. **App consent UI accepted** → `controller.grantBaseline()` grants the
   first-party `analytics` category and flips the persistence predicate; now
   (and only now) `Identity`/`Session` persist via the secure store. Sinks
   declaring `['analytics']` (Firebase/PostHog defaults) start receiving.
3. **iOS ATT** — the consuming app shows the system prompt **after first
   frame** (Apple HIG) by calling `controller.requestTracking()` (this
   library **never** auto-prompts). On `authorized` the `tracking` category
   is granted; on `denied`/`restricted` it is revoked, so any
   tracking-category sink stays blocked by the core gate. Off-iOS the status
   is `notApplicable` and the app governs `tracking` via explicit `grant`.
4. **Withdrawal** — `controller.revokeAll()` revokes every category, clears
   the persistence consent, and purges the persisted identity from the secure
   store (privacy-safe; nothing re-mints until consent is granted again).
   `analytics.reset()` independently rotates to a fresh non-PII id.

App-background session boundary (GA4 parity): the analytics session ends after
the app is backgrounded ≥ 30 min (`AppBackgroundSessionPolicy`/`Binder`,
driven by the `AppLifecycleHook` surface) — no identifiers involved.

## 5. Consumer wiring (what the app adds — these are real native deps)

The library bundles none of these; the app adds the ones it uses and writes a
~5-line adapter onto the injectable port. **Each is a genuine native plugin**
(Gradle/CocoaPods + MethodChannels) and the consuming app — not
`refraction_ui` — owns its store-listing declarations:

| Port (this layer) | Consumer's native plugin (added by the app) |
|---|---|
| `FirebaseAnalyticsClient` | `firebase_core` + `firebase_analytics` (+ `GoogleService-Info.plist` / `google-services.json`) |
| `PostHogClient` | `posthog_flutter` |
| `SecureKeyValueStore` | `flutter_secure_storage` (Keychain/Keystore) or `shared_preferences` |
| `TrackingAuthorization` | `app_tracking_transparency` (iOS ATT) |
| `AppLifecycleHook` | none — uses Flutter's built-in `WidgetsBinding` (already a transitive dep; not a new pub package) |

This injectable-port design is deliberate (CLAUDE.md: *minimize pub deps*; the
package has zero runtime deps) and mirrors the web epic keeping `posthog-js`
an optional, never-bundled peer. It keeps the public API and package
structure **identical across Flutter web/Android/iOS/desktop**.
